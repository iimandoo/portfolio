🟢 **Phase 1 — 콘텐츠 설정** (필수)

포트폴리오에 표시될 모든 콘텐츠를 `data/resume.ts` 단일 파일로 관리합니다.

⚠️ **사전 조건**: `/project-init` 명령어가 먼저 실행되어야 합니다.

## 목표

- `data/resume.ts` 를 수정 후 `/sample-page-create` 로 page.tsx를 재생성하면 전체 반영

## Steps

### 1. `data/resume.ts` 존재 여부 확인 (필수)

Read 툴로 `data/resume.ts`를 읽습니다.

- **파일이 있으면 → Step 2로 진행**
- **파일이 없으면 → 작업을 즉시 중단**하고 아래 메시지를 출력:

```
❌ data/resume.ts 파일을 찾을 수 없습니다.

먼저 /project-init 을 실행하세요.
```

---

### 2. `data/resume.ts` 구조 검증 및 보완

읽은 파일의 최상위 섹션을 확인합니다. 아래 섹션 중 누락된 항목이 있으면 파일 끝(`) as const` 직전)에 추가합니다.

| 섹션             | 키        | 필수 여부 |
| ---------------- | --------- | --------- |
| SEO & 메타데이터 | `meta`    | 필수      |
| 기본 프로필      | `profile` | 필수      |
| 경력 타임라인    | `career`  | 필수      |
| 기술 스택        | `skills`  | 필수      |
| 프로젝트         | `Project` | 필수      |

누락된 섹션은 아래 **전체 구조 레퍼런스**의 샘플 데이터로 채웁니다.

---

## Rules

- **단일 소스**: 콘텐츠 수정은 오직 `data/resume.ts` 하나만 변경 (portfolio-agent 루트)
- **JSON 금지**: 별도 JSON 파일 생성하지 않을 것
- **파일 없으면 중단**: `data/resume.ts`가 없으면 작업을 시작하지 않음 (`/project-init` 선행 필수)
- **덮어쓰기 금지**: 기존 파일의 내용을 덮어쓰지 않고 누락 섹션만 추가
- **타입 안정성**: `as const` + 추출 타입으로 컴파일 타임 검증
- **이미지 경로**: 모두 `public/images/` 상대 경로 사용
- **임포트 금지**: 컴포넌트에서 `data/resume.ts`를 직접 import하지 않음 — `/sample-page-create`로 page.tsx에 하드코딩
