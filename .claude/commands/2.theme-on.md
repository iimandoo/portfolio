# 🎨 Theme Manager — 개별 테마 추가 설치

> ⚠️ `/project-init`은 이미 모든 Style × Tone 조합을 자동 생성합니다.
> 이 명령어는 **새 스타일/톤이 추가되었거나 특정 조합만 재설치**할 때 사용합니다.

---

## 동작 분기 ($ARGUMENTS 기준)

### `$ARGUMENTS`가 **`{style}/{tone}`** 형식인 경우

예: `/theme-on block/toss`, `/theme-on corporate/kakao`

**실행 순서:**

1. **공용 인프라 확인** — `src/styles/provider.tsx`와 `src/styles/styled.d.ts`가 없으면 생성합니다 (1.project-init.md의 3-1 섹션 참조).
2. **Tone 토큰 설치** — `.claude/commands/3[a-z]*.md`에서 tone을 찾아 `src/styles/themes/{tone}.ts` 생성.
3. **스타일 컴포넌트 확인** — `src/components/{style}/`이 없으면 `2[a-z]*.md` 파일을 읽어 생성.
4. **Route layout 생성** — `src/app/{style}/{tone}/layout.tsx`:
   ```typescript
   'use client';
   import { StyleProvider } from '@/styles/provider';
   import { {tone}Theme } from '@/styles/themes/{tone}';
   export default function Layout({ children }: { children: React.ReactNode }) {
     return <StyleProvider theme={{tone}Theme}>{children}</StyleProvider>;
   }
   ```
5. **page.tsx 생성** — 해당 스타일의 page.tsx를 `src/app/{style}/{tone}/page.tsx`에 생성.
6. **랜딩 페이지 업데이트** — `src/app/page.tsx`의 routes 배열에 새 조합 추가.
7. **빌드 확인** — `cd c:/work/jione-portfolio && npm run build`

### `$ARGUMENTS`가 **`all`**인 경우

`/project-init`과 동일하게 `.claude/commands/2[a-z]*.md`와 `3[a-z]*.md`를 전체 스캔하여 모든 조합을 재설치합니다.

### `$ARGUMENTS`가 **`status`**인 경우

현재 설치 상태를 점검합니다:

1. 공용 인프라 파일 존재: `provider.tsx`, `styled.d.ts`, `globals.css`
2. `2[a-z]*.md` → 사용 가능한 Style 목록
3. `3[a-z]*.md` → 사용 가능한 Tone 목록
4. `src/app/{style}/{tone}/layout.tsx` 존재 여부

### `$ARGUMENTS`가 **비어 있거나 없는** 경우

```
어떤 조합을 추가할까요?

📐 Style:
  1) block      — 카드 기반 섹션 (Hero → About → Projects → Contact)
  2) corporate  — 기업 사이트형 (GNB + 대형 Hero + 카드 슬라이더)

🎨 Tone:
  A) toss    — Toss Blue, 둥근 모서리
  B) kakao   — 카카오 옐로우, 다크 네비

예) /theme-on block/toss  또는  /theme-on corporate/kakao
```

## 공통 Rules

- `provider.tsx`, `styled.d.ts` 이미 존재 시 덮어쓰지 말고 필요한 부분만 업데이트
- 모든 컴포넌트: `props.theme.colors.*`, `props.theme.radius.*`, `props.theme.shadows.*` 참조
- 빌드 성공 필수: `cd c:/work/jione-portfolio && npm run build`
