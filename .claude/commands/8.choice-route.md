🟢 **Phase 1 — Choice Route 설치**

`/choice` 에서 Style(block/corporate)과 Tone(toss/minimal/dark/kakao)을 선택하면
`/` 랜딩 페이지에 해당 조합이 표시된다.
선택값은 `localStorage` 에 저장되며, 최초 방문 시 기본값은 **block + toss**.

---

## 핵심 설계

| 항목 | 설명 |
|------|------|
| 저장소 | `localStorage` — `portfolio-style`, `portfolio-tone` 키 |
| 기본값 | style: `block`, tone: `toss` |
| `/choice` | 8개 조합 선택 UI → 선택 시 저장 후 `/` 로 이동 |
| `/` | 저장된 값 읽기 → StyleProvider + 해당 스타일 컴포넌트 렌더 |
| 설정 버튼 | `/` 우상단에 ⚙ 버튼 → `/choice` 로 이동 |

## Steps

### Step 1: `src/lib/portfolio-selection.ts` 유틸 생성

```typescript
export const STYLES = ['block', 'corporate'] as const;
export const TONES = ['toss', 'minimal', 'dark', 'kakao'] as const;

export type Style = (typeof STYLES)[number];
export type Tone = (typeof TONES)[number];

const STYLE_KEY = 'portfolio-style';
const TONE_KEY = 'portfolio-tone';

export function getSelection(): { style: Style; tone: Tone } {
  if (typeof window === 'undefined') return { style: 'block', tone: 'toss' };
  const style = (localStorage.getItem(STYLE_KEY) as Style) || 'block';
  const tone = (localStorage.getItem(TONE_KEY) as Tone) || 'toss';
  return { style, tone };
}

export function setSelection(style: Style, tone: Tone) {
  localStorage.setItem(STYLE_KEY, style);
  localStorage.setItem(TONE_KEY, tone);
}
```

### Step 2: `src/app/choice/page.tsx` 선택 페이지 생성

- `'use client'` 필수
- 8개 조합(2 Style × 4 Tone) 을 카드 그리드로 표시
- 각 카드에 Style 이름, Tone 이름, Tone 대표색 표시
- 클릭 시 `setSelection(style, tone)` 호출 후 `router.push('/')`
- 현재 선택된 조합에 체크마크 또는 하이라이트 표시
- 스타일링: styled-components 사용 금지 — inline style 또는 globals.css 클래스 사용 (테마 불필요 페이지)
- 레이아웃: 상단 제목 "포트폴리오 스타일 선택" + 설명 + 2열 그리드 (Style별 그룹)

**Tone 대표색 참조:**

| Tone | Primary | 배경 힌트 |
|------|---------|-----------|
| toss | `#3182F6` | `#F2F4F6` |
| minimal | `#111111` | `#FAFAFA` |
| dark | `#60A5FA` | `#0F172A` |
| kakao | `#FEE500` | `#F7F2E8` |

**Style 설명:**

| Style | 설명 |
|-------|------|
| block | 카드 기반 섹션 — Hero → About → Projects → Contact |
| corporate | GNB + 대형 Hero + 카드 슬라이더 — 기업 사이트 스타일 |

### Step 3: `src/app/page.tsx` 랜딩 페이지 교체

기존 정적 선택 페이지를 **동적 렌더링 페이지**로 교체한다.

**동작:**
1. `useEffect`로 `getSelection()` 호출 → `style`, `tone` 상태 세팅
2. 테마 맵에서 tone에 해당하는 theme 객체 가져오기
3. `<StyleProvider theme={theme}>` 으로 감싸기
4. style에 따라 block 또는 corporate 컴포넌트 렌더링
5. 우상단에 설정 버튼(⚙) 고정 → 클릭 시 `/choice` 이동

**필요 import:**

```typescript
// themes
import { tossTheme } from '@/styles/themes/toss';
import { minimalTheme } from '@/styles/themes/minimal';
import { darkTheme } from '@/styles/themes/dark';
import { kakaoTheme } from '@/styles/themes/kakao';

// block components
import { HeroSection } from '@/components/block/hero';
import { AboutSection } from '@/components/block/about';
import { ProjectSection } from '@/components/block/projects';
import { ContactSection } from '@/components/block/contact';

// corporate components — 현재 page.tsx 에서 props 전달 방식 참조
// ⚠️ corporate 컴포넌트는 props를 받는 형태일 수 있음
// → 기존 corporate/{tone}/page.tsx 의 props 전달 패턴을 그대로 따른다
```

**테마 맵:**

```typescript
const themeMap = {
  toss: tossTheme,
  minimal: minimalTheme,
  dark: darkTheme,
  kakao: kakaoTheme,
} as const;
```

**설정 버튼 (⚙) 스타일:**
- `position: fixed`, `top: 1rem`, `right: 1rem`, `z-index: 1000`
- 원형 버튼, 배경 반투명, 클릭 시 `/choice` 이동
- Block 스타일일 때 styled.button 사용 가능 (StyleProvider 내부)

**Corporate 컴포넌트 props 처리:**
- 기존 `src/app/corporate/{tone}/page.tsx` 파일을 Read하여 props 전달 패턴 확인
- 동일한 데이터/props를 랜딩 페이지에서도 전달

**로딩 처리:**
- `loaded` 상태가 false일 때 빈 화면 또는 간단한 로딩 표시
- hydration mismatch 방지를 위해 서버에서는 아무것도 렌더하지 않음

## Rules

- `/choice` 페이지는 테마 없이 동작 (styled-components 미사용)
- `/` 페이지는 `'use client'` 필수
- corporate 컴포넌트의 props 패턴은 기존 page.tsx를 반드시 참조하여 동일하게 전달
- block 컴포넌트는 resume.ts에서 직접 import하므로 props 불필요
- `getSelection()` / `setSelection()` 유틸을 반드시 사용 (직접 localStorage 접근 금지)
- hydration 에러 방지: useEffect 내에서만 localStorage 접근
