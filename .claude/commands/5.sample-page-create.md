🟡 **유틸리티 — 특정 페이지 재생성**

특정 Style × Tone 조합의 `page.tsx`를 **data/resume.ts 값을 하드코딩**하여 재생성합니다.
`/project-init`은 이미 모든 페이지를 생성합니다. 이 명령어는 **특정 조합만 덮어쓰거나 복구**할 때 사용합니다.

⚠️ **사전 조건**: `/project-init` → `/content-setup` 실행 완료

---

## 아키텍처 원칙

- **컴포넌트는 props를 받는다** — `import { resume }` 없음
- **page.tsx가 데이터를 하드코딩** — `data/resume.ts`를 읽어 값을 그대로 적어넣음
- **data/resume.ts는 commands의 소스** — 런타임 import 대상이 아님
- data/resume.ts를 수정하면 반드시 이 명령어로 page.tsx를 재생성해야 반영됨

---

## Step 1: data/resume.ts 읽기

`data/resume.ts`를 **반드시 Read 툴로 읽어** 모든 값을 파악합니다.

파일이 없으면 중단하고 `/content-setup` 먼저 실행 안내.

읽은 데이터에서 다음 값을 추출합니다:
- `resume.profile` — name, title, subtitle, bio, email, phone, location, availability, social.github
- `resume.career.experiences` — 전체 배열
- `resume.skills.categories` — 전체 배열
- `resume.Project.intro` — 프로젝트 소개 문구
- `resume.Project.cases` — 전체 배열 (urls 포함)

---

## Step 2: page.tsx 생성 (하드코딩 방식)

읽은 resume.ts의 **실제 값**을 page.tsx 안에 직접 기입합니다.
템플릿 변수나 import 없이, 리터럴 문자열/배열로 작성합니다.

### Block 스타일 page.tsx 구조

```typescript
// src/app/block/{tone}/page.tsx
'use client';

import styled from 'styled-components';
import { HeroSection } from '@/components/block/hero';
import { AboutSection } from '@/components/block/about';
import { ProjectsSection } from '@/components/block/projects';
import { ContactSection } from '@/components/block/contact';

const Main = styled.main`
  width: 100%;
  overflow-x: hidden;
`;

// ─── 콘텐츠 데이터 (data/resume.ts 기준) ──────────────────────────
// data/resume.ts 를 수정한 경우 /sample-page-create 명령어로 재생성하세요.

const profile = {
  name: '{resume.profile.name}',
  title: '{resume.profile.title}',
  subtitle: '{resume.profile.subtitle}',
  bio: '{resume.profile.bio}',
  email: '{resume.profile.email}',
  phone: '{resume.profile.phone}',
  github: '{resume.profile.social.github}',
};

const experiences = [
  // resume.career.experiences 배열을 그대로 하드코딩
  { id: '...', company: '...', position: '...', period: { start: '...', end: '...' }, description: '...' },
  // ...
];

const skillCategories = [
  // resume.skills.categories 배열을 그대로 하드코딩
  { name: '...', skills: [{ name: '...', level: '...' }, ...] },
  // ...
];

const projectIntro = '{resume.Project.intro}';

const projectCases = [
  // resume.Project.cases 배열을 그대로 하드코딩
  // urls 필드가 있으면 포함, 없으면 생략
  { id: '...', title: '...', company: '...', role: '...', description: '...', execution: '...', tools: '...', year: '...', images: [...] },
  { id: '...', title: '...', company: '...', role: '...', urls: [{ label: '...', href: '...' }], images: [...] },
  // ...
];

export default function Page() {
  return (
    <Main>
      <HeroSection
        name={profile.name}
        title={profile.title}
        subtitle={profile.subtitle}
        bio={profile.bio}
      />
      <AboutSection
        experiences={experiences}
        skillCategories={skillCategories}
      />
      <ProjectsSection
        intro={projectIntro}
        cases={projectCases}
      />
      <ContactSection
        email={profile.email}
        phone={profile.phone}
        github={profile.github}
      />
    </Main>
  );
}
```

## Step 3: 컴포넌트 props 인터페이스 확인

컴포넌트가 props를 받도록 작성되어 있는지 확인합니다.

### block 컴포넌트 props 요약

| 컴포넌트 | Props |
|----------|-------|
| `HeroSection` | `name, title, subtitle, bio` |
| `AboutSection` | `experiences[], skillCategories[]` |
| `ProjectsSection` | `intro, cases[]` (cases에 `urls?` 포함) |
| `ContactSection` | `email, phone?, github?` |

### corporate 컴포넌트 props 요약

| 컴포넌트 | Props |
|----------|-------|
| `GNB` | `name, github, email` |
| `Hero` | `title, name, subtitle, location?, availability?` |
| `CardSlider` | `cases[]` (cases에 `urls?` 포함) |
| `Career` | `experiences[], skillCategories[]` |
| `Contact` | `name, bio, email, phone?, github?` |

컴포넌트가 여전히 `import { resume }` 방식이라면 props 방식으로 교체합니다.

## Rules

- **data/resume.ts → 런타임 import 금지** — page.tsx에서 `import { resume }` 사용하지 않음
- **값은 반드시 하드코딩** — 읽은 resume.ts의 실제 값을 리터럴로 기입
- **urls 필드**: cases에 urls 배열이 있으면 그대로 포함, 없으면 생략
- 모든 스타일은 `styled-components`, `props.theme.*` 토큰 사용
- 이미지는 `next/image`, 애니메이션은 Framer Motion
- 생성 후 반드시 `npm run build` 실행하여 에러 확인
