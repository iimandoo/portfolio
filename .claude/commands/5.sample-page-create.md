🟡 **유틸리티 — 특정 페이지 재생성**

특정 Style × Tone 조합의 컴포넌트와 `page.tsx`를 재생성합니다.
`/project-init`은 이미 모든 페이지를 생성합니다. 이 명령어는 **특정 조합만 덮어쓰거나 복구**할 때 사용합니다.

⚠️ **사전 조건**: `/project-init` → `/content-setup` 실행 완료

---

## Step 0: 조합 선택

**`$ARGUMENTS`가 없는 경우**, 설치된 라우트를 확인하고 사용자에게 선택을 요청합니다:

```bash
# 설치된 라우트 확인
ls jione-portfolio/src/app/block/ 2>/dev/null
ls jione-portfolio/src/app/corporate/ 2>/dev/null
ls jione-portfolio/src/styles/themes/
```

확인 후 아래 옵션을 제시합니다:

```
어떤 조합으로 페이지를 생성할까요?

📐 Style:
  block     — 카드 기반 (Hero → About → Projects → Contact)
  corporate — 기업 사이트형 (GNB + Hero + CardSlider)

🎨 Tone (설치된 것만 표시):
  toss / kakao

예) block/toss, corporate/kakao

※ 해당 조합이 미설치라면 먼저 /theme-on {style}/{tone} 을 실행하세요.
```

사용자 선택에 따라 아래 변수를 결정합니다:

| 변수 | 예시 |
|------|------|
| `TARGET_STYLE` | `block` 또는 `corporate` |
| `TARGET_TONE` | `toss`, `kakao` |
| `TARGET_ROUTE` | `block/toss` |
| `PAGE_PATH` | `src/app/block/toss/page.tsx` |
| `COMPONENTS_DIR` | `src/components/block/` 또는 `src/components/corporate/` |

---

## 사전 체크

1. `jione-portfolio/src/styles/themes/{TARGET_TONE}.ts` — 존재해야 함
2. `jione-portfolio/src/styles/provider.tsx` — StyleProvider
3. `jione-portfolio/src/app/{TARGET_ROUTE}/layout.tsx` — ThemeProvider 래퍼
4. `jione-portfolio/data/resume.ts` — 콘텐츠 소스 (`@data/resume` 로 임포트)

---

## Steps

### 1. 스타일 명령 파일 읽기

`.claude/commands/2[a-z]*.md` 파일에서 `TARGET_STYLE`과 매칭되는 파일을 읽습니다.
(예: `block` → `2a.style-block.md`, `corporate` → `2b.style-corporate.md`)

해당 파일의 Steps에 정의된 **컴포넌트 코드를 그대로 `COMPONENTS_DIR`에 생성**합니다.

> 컴포넌트가 이미 존재하면 덮어쓰지 않고 내용을 비교 후 필요한 부분만 업데이트합니다.

### 2. `src/app/{TARGET_ROUTE}/page.tsx` 생성

**Block 스타일일 경우:**

```typescript
// src/app/block/{tone}/page.tsx
'use client';

import styled from 'styled-components';
import { HeroSection } from '@/components/block/hero';
import { AboutSection } from '@/components/block/about';
import { ProjectsSection } from '@/components/block/projects';
import { ContactSection } from '@/components/block/contact';

const Main = styled.main`
  width: 100%;
  overflow-x: hidden;
`;

export default function Page() {
  return (
    <Main>
      <HeroSection />
      <AboutSection />
      <ProjectsSection />
      <ContactSection />
    </Main>
  );
}
```

**Corporate 스타일일 경우:**

```typescript
// src/app/corporate/{tone}/page.tsx
'use client';

import { GNB } from '@/components/corporate/gnb';
import { Hero } from '@/components/corporate/hero';
import { CardSlider } from '@/components/corporate/card-slider';
import { Career } from '@/components/corporate/career';
import { Contact } from '@/components/corporate/contact';

export default function Page() {
  return (
    <>
      <GNB />
      <Hero />
      <CardSlider />
      <Career />
      <Contact />
    </>
  );
}
```

### 3. Material Symbols 아이콘 폰트 추가

`jione-portfolio/src/app/layout.tsx`의 `<head>`에 CDN 링크가 없으면 추가합니다:

```html
<link
  href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined:opsz,wght,FILL,GRAD@20..48,100..700,0..1,-50..200"
  rel="stylesheet"
/>
```

### 4. 빌드 및 검증

```bash
cd jione-portfolio && npm run build
npm run dev
```

`http://localhost:3000/{TARGET_ROUTE}` 에서 확인합니다.

---

## Rules

- 컴포넌트가 이미 존재하면 덮어쓰지 않고 내용 비교 후 필요 부분만 업데이트
- 모든 스타일은 `styled-components`, `props.theme.*` 토큰 사용
- 이미지는 `next/image`
- 애니메이션은 Framer Motion (`whileInView` + `viewport={{ once: true }}`)
- 생성 후 반드시 `npm run build` 실행하여 에러 확인
