🟢 **Phase 1 — 디자인 시스템** (필수)

포트폴리오 프로젝트에 토스 및 카카오 스타일을 모두 준비하고, 사용자
가 런타임에 테마를 전환하거나 ‘모두’ 보기 대시보드를 열 수 있는 기능을
추가합니다. 기본 실행 시 색상 토큰은 토스 블루로 세팅되며, 나중에
`ThemeSwitcher`를 통해 떨궈 선택 가능합니다.

⚠️ **사전 조건**: `/project-init` 명령어가 먼저 실행되어야 합니다.

## 사전 체크

아래 파일을 Read 툴로 읽어 현재 상태를 파악하세요:

- `jione-portfolio/src/styles/globals.css` — 현재 CSS (리셋 및 기본 스타일)
- `jione-portfolio/src/styles/theme.ts` — 디자인 토큰 (styled-components 테마)
- `jione-portfolio/src/styles/provider.tsx` — ThemeProvider 설정

## 디자인 토큰 (styled-components 테마)

두 가지 스타일(토스 / 카카오)을 모두 지원하기 위해 `theme.ts`에는
`themeToss`와 `themeKakao`라는 두 개의 객체를 내보냅니다. 구조는 동일하기
때문에 TypeScript 타입은 하나로 재사용할 수 있습니다.

### 토스 컬러 토큰 (예시)

```typescript
// src/styles/theme.ts
export const themeToss = {
  colors: {
    background: "#FFFFFF",
    foreground: "#1B1C1F",
    primary: "#3182F6", // Toss Blue (또는 $ARGUMENTS로 커스텀 가능)
    primaryForeground: "#FFFFFF",
    secondary: "#F2F4F6",
    secondaryForeground: "#4E5968",
    muted: "#F9FAFB",
    mutedForeground: "#8B95A1",
    border: "#E5E8EB",
    card: "#FFFFFF",
    cardForeground: "#1B1C1F",
    destructive: "#FF4D4F",
  },
  // ...radius, shadows, fonts, typography 동일
};
```

### 카카오 컬러 토큰 (예시)

```typescript
export const themeKakao = {
  colors: {
    background: "#FFFFFF",
    foreground: "#1A1A1A",
    primary: "#FEE500", // Kakao Yellow
    primaryForeground: "#000000",
    secondary: "#F6F6F6",
    secondaryForeground: "#3C1E1E",
    muted: "#FAFAFA",
    mutedForeground: "#8C8C8C",
    border: "#E0E0E0",
    card: "#FFFFFF",
    cardForeground: "#1A1A1A",
    destructive: "#E60023",
  },
  // ...radius, shadows, fonts, typography 동일 형식
};
```

실제 파일에는 위 두 객체를 모두 정의한 뒤

```typescript
export const themes = { toss: themeToss, kakao: themeKakao } as const;
export type Theme = typeof themeToss;
export type ThemeName = keyof typeof themes;
```

이와 같이 두 가지 스타일을 함께 준비해 두면 런타임에서 스위칭이
가능하며 ‘모두 보기’ 기능도 구현할 수 있습니다.

## Steps

1. **Theme 토큰 정의**
   - `src/styles/theme.ts` 생성 및 위 토큰 작성
   - TypeScript로 타입 안정성 확보

2. **ThemeProvider 생성**
   - `src/styles/provider.tsx` 생성
   - styled-components의 `ThemeProvider` 설정
   - Root Layout에 감싸기

3. **styled-components 타입 확장**
   - `src/styles/styled.d.ts` 생성 (DefaultTheme 타입 확장 필수)
   - 이 파일이 없으면 `props.theme.colors.*` 사용 시 TypeScript 오류 발생

4. **globals.css 정리**
   - CSS 리셋 및 기본 스타일만 정의

5. **컴포넌트 스타일 작성**
   - 각 섹션/컴포넌트를 styled-components로 작성
   - 토스 디자인 토큰 참조

6. **빌드 확인**
   ```bash
   cd jione-portfolio && npm run build
   ```
   빌드 에러가 있으면 수정 후 재시도

### 상세 구현

#### Step 1: `src/styles/theme.ts` 생성

위 토큰 섹션에 설명한 대로 `themeToss`와 `themeKakao` 두 객체를 모두 정의하고
`themes` 객체로 묶어 내보내세요. 토스 컬러는 `$ARGUMENTS`로 커스터마이징 할
수 있습니다. 결과적으로 타입은 다음과 같이 나타납니다.

```typescript
import { themeToss, themeKakao } from "./theme";

export const themes = { toss: themeToss, kakao: themeKakao } as const;
export type Theme = typeof themeToss; // 두 테마는 동일한 구조
export type ThemeName = keyof typeof themes;
```

이 파일을 만들어 두면 이후 provider와 switcher에서 이름 기반으로
테마를 선택할 수 있습니다.

#### Step 2: `src/styles/styled.d.ts` 생성 (필수 — TypeScript 타입 확장)

```typescript
import "styled-components";
import { Theme } from "./theme";

declare module "styled-components" {
  export interface DefaultTheme extends Theme {}
}
```

> ⚠️ 이 파일이 없으면 컴포넌트에서 `props.theme.colors.*` 사용 시 TypeScript 빌드 오류가 발생합니다.

#### Step 3: `src/styles/provider.tsx` 생성

이제 테마를 동적으로 전환할 수 있도록 Context와 상태를 추가합니다.
`'all'` 옵션을 선택하면 두 스타일을 나란히 렌더링하는 대시보드를
만들어 줍니다. 코드는 다음과 같습니다 (파일에 그대로 붙여넣어도 됩니다).

```typescript
'use client';

import React, { createContext, useContext, useEffect, useState } from 'react';
import { useServerInsertedHTML } from 'next/navigation';
import { ServerStyleSheet, StyleSheetManager, ThemeProvider } from 'styled-components';
import { themes, ThemeName } from './theme';

// Theme 옵션 타입 (토스/카카오/모두)
type ThemeOption = ThemeName | 'all';

interface ThemeContextType {
  option: ThemeOption;
  setOption: (opt: ThemeOption) => void;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

export function useTheme() {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error('useTheme must be used within ThemeContext');
  return ctx;
}

export function StyleProvider({ children }: { children: React.ReactNode }) {
  const [styledComponentsStyleSheet] = useState(() => new ServerStyleSheet());
  const [option, setOption] = useState<ThemeOption>(() => {
    if (typeof window === 'undefined') return 'toss';
    return (localStorage.getItem('themeOption') as ThemeOption) || 'toss';
  });

  useEffect(() => {
    if (typeof window !== 'undefined') {
      localStorage.setItem('themeOption', option);
    }
  }, [option]);

  useServerInsertedHTML(() => {
    const styles = styledComponentsStyleSheet.getStyleElement();
    styledComponentsStyleSheet.instance.clearTag();
    return <>{styles}</>;
  });

  const renderWithTheme = (theme: typeof themes.toss, child: React.ReactNode) => (
    <ThemeProvider theme={theme}>{child}</ThemeProvider>
  );

  const content =
    option === 'all' ? (
      <div className="theme-dashboard">
        {Object.entries(themes).map(([name, t]) => (
          <div className="theme-pane" key={name}>
            <h2 style={{ textAlign: 'center', marginBottom: '0.5rem' }}>{name}</h2>
            {renderWithTheme(t, children)}
          </div>
        ))}
      </div>
    ) : (
      renderWithTheme(themes[option as ThemeName], children)
    );

  const providerTree =
    typeof window !== 'undefined' ? (
      content
    ) : (
      <StyleSheetManager sheet={styledComponentsStyleSheet.instance}>
        {content}
      </StyleSheetManager>
    );

  return (
    <ThemeContext.Provider value={{ option, setOption }}>
      {providerTree}
    </ThemeContext.Provider>
  );
}
```

## Rules

- **테마 일관성**: 모든 컴포넌트는 `styled-components`와 `theme` 객체 사용
- **기존 콘텐츠**: 텍스트, 이미지 경로, data.ts는 절대 변경하지 않음
- **빌드 성공**: 반드시 `npm run build` 실행 후 에러 없음 확인
- **TypeScript**: 타입 안정성을 위해 `styled.ts` 파일 생성 시 타입 정의 필수
- **반응형**: mobile-first 방식으로 media queries 작성
