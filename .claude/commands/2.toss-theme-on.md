🟢 **Phase 1 — 디자인 시스템 · Toss 테마** (필수)

`/toss` 라우트에 Toss 스타일 디자인 토큰을 적용합니다.

⚠️ **사전 조건**: `/project-init` 명령어가 먼저 실행되어야 합니다.

## 사전 체크

아래 파일들을 Read 툴로 읽어 현재 상태를 파악하세요:

- `jione-portfolio/src/styles/globals.css`
- `jione-portfolio/src/styles/provider.tsx` (없으면 신규 생성)
- `jione-portfolio/src/styles/styled.d.ts` (없으면 신규 생성)

## 생성할 파일

이 명령어는 아래 파일들을 생성합니다:

| 파일 | 역할 |
|------|------|
| `src/styles/themes/toss.ts` | Toss 디자인 토큰 |
| `src/styles/provider.tsx` | theme prop을 받는 공용 StyleProvider |
| `src/styles/styled.d.ts` | DefaultTheme 타입 확장 |
| `src/styles/globals.css` | 전역 CSS 리셋 |
| `src/app/toss/layout.tsx` | Toss ThemeProvider 래퍼 |

### Step 2: `src/styles/styled.d.ts` 생성 (DefaultTheme 타입 확장)

### Step 3: `src/styles/provider.tsx` 생성 (공용 StyleProvider)

이 파일은 **모든 테마가 공유**하는 SSR 스타일 주입 + ThemeProvider 래퍼입니다.

```typescript
'use client';

import React, { useState } from 'react';
import { useServerInsertedHTML } from 'next/navigation';
import { ServerStyleSheet, StyleSheetManager, ThemeProvider } from 'styled-components';
import type { DefaultTheme } from 'styled-components';

// SSR 스타일 주입 (FOUC 방지) — Next.js App Router + styled-components v6 필수
export function StyleProvider({
  children,
  theme,
}: {
  children: React.ReactNode;
  theme: DefaultTheme;
}) {
  const [styledComponentsStyleSheet] = useState(() => new ServerStyleSheet());

  useServerInsertedHTML(() => {
    const styles = styledComponentsStyleSheet.getStyleElement();
    styledComponentsStyleSheet.instance.clearTag();
    return <>{styles}</>;
  });

  if (typeof window !== 'undefined') {
    return <ThemeProvider theme={theme}>{children}</ThemeProvider>;
  }

  return (
    <StyleSheetManager sheet={styledComponentsStyleSheet.instance}>
      <ThemeProvider theme={theme}>{children}</ThemeProvider>
    </StyleSheetManager>
  );
}
```

## Rules

- **테마 일관성**: 모든 컴포넌트는 `props.theme.colors.*`, `props.theme.radius.*` 참조
- **provider.tsx**: 이미 존재하면 덮어쓰지 말고 내용을 비교 후 필요한 부분만 업데이트
- **styled.d.ts**: 이미 존재하면 import 경로 확인 후 필요 시 업데이트
- **빌드 성공**: 반드시 `npm run build` 실행 후 에러 없음 확인
