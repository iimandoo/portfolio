🟢 **Phase 1 — 디자인 시스템 · Toss 테마** (필수)

`/toss` 라우트에 Toss 스타일 디자인 토큰을 적용합니다.

⚠️ **사전 조건**: `/project-init` 명령어가 먼저 실행되어야 합니다.

## 사전 체크

아래 파일들을 Read 툴로 읽어 현재 상태를 파악하세요:

- `jione-portfolio/src/styles/globals.css`
- `jione-portfolio/src/styles/provider.tsx` (없으면 신규 생성)
- `jione-portfolio/src/styles/styled.d.ts` (없으면 신규 생성)

## 생성할 파일

이 명령어는 아래 파일들을 생성합니다:

| 파일                        | 역할                                 |
| --------------------------- | ------------------------------------ |
| `src/styles/themes/toss.ts` | Toss 디자인 토큰                     |
| `src/styles/provider.tsx`   | theme prop을 받는 공용 StyleProvider |
| `src/styles/styled.d.ts`    | DefaultTheme 타입 확장               |
| `src/styles/globals.css`    | 전역 CSS 리셋                        |
| `src/app/toss/layout.tsx`   | Toss ThemeProvider 래퍼              |

### 카카오 컬러 토큰 (예시)

```typescript
export const themeKakao = {
  colors: {
    background: "#FFFFFF",
    foreground: "#1A1A1A",
    primary: "#FEE500", // Kakao Yellow
    primaryForeground: "#000000",
    secondary: "#F6F6F6",
    secondaryForeground: "#3C1E1E",
    muted: "#FAFAFA",
    mutedForeground: "#8C8C8C",
    border: "#E0E0E0",
    card: "#FFFFFF",
    cardForeground: "#1A1A1A",
    destructive: "#E60023",
  },
  // ...radius, shadows, fonts, typography 동일 형식
};
```

실제 파일에는 위 두 객체를 모두 정의한 뒤

```typescript
export const themes = { toss: themeToss, kakao: themeKakao } as const;
export type Theme = typeof themeToss;
export type ThemeName = keyof typeof themes;
```

이와 같이 두 가지 스타일을 함께 준비해 두면 런타임에서 스위칭이
가능하며 ‘모두 보기’ 기능도 구현할 수 있습니다.

### Step 3: `src/styles/provider.tsx` 생성 (공용 StyleProvider)

이 파일은 **모든 테마가 공유**하는 SSR 스타일 주입 + ThemeProvider 래퍼입니다.

이제 테마를 동적으로 전환할 수 있도록 Context와 상태를 추가합니다.
`'all'` 옵션을 선택하면 두 스타일을 나란히 렌더링하는 대시보드를
만들어 줍니다. 코드는 다음과 같습니다 (파일에 그대로 붙여넣어도 됩니다).

```typescript
'use client';

import React, { createContext, useContext, useEffect, useState } from 'react';
import { useServerInsertedHTML } from 'next/navigation';
import { ServerStyleSheet, StyleSheetManager, ThemeProvider } from 'styled-components';
import type { DefaultTheme } from 'styled-components';

// SSR 스타일 주입 (FOUC 방지) — Next.js App Router + styled-components v6 필수
export function StyleProvider({
  children,
  theme,
}: {
  children: React.ReactNode;
  theme: DefaultTheme;
}) {
  const [styledComponentsStyleSheet] = useState(() => new ServerStyleSheet());
  const [option, setOption] = useState<ThemeOption>(() => {
    if (typeof window === 'undefined') return 'toss';
    return (localStorage.getItem('themeOption') as ThemeOption) || 'toss';
  });

  useEffect(() => {
    if (typeof window !== 'undefined') {
      localStorage.setItem('themeOption', option);
    }
  }, [option]);

  useServerInsertedHTML(() => {
    const styles = styledComponentsStyleSheet.getStyleElement();
    styledComponentsStyleSheet.instance.clearTag();
    return <>{styles}</>;
  });

  const renderWithTheme = (theme: typeof themes.toss, child: React.ReactNode) => (
    <ThemeProvider theme={theme}>{child}</ThemeProvider>
  );

  const content =
    option === 'all' ? (
      <div className="theme-dashboard">
        {Object.entries(themes).map(([name, t]) => (
          <div className="theme-pane" key={name}>
            <h2 style={{ textAlign: 'center', marginBottom: '0.5rem' }}>{name}</h2>
            {renderWithTheme(t, children)}
          </div>
        ))}
      </div>
    ) : (
      renderWithTheme(themes[option as ThemeName], children)
    );

  const providerTree =
    typeof window !== 'undefined' ? (
      content
    ) : (
      <StyleSheetManager sheet={styledComponentsStyleSheet.instance}>
        {content}
      </StyleSheetManager>
    );

  return (
    <ThemeContext.Provider value={{ option, setOption }}>
      {providerTree}
    </ThemeContext.Provider>
  );
}
```

## Rules

- **테마 일관성**: 모든 컴포넌트는 `props.theme.colors.*`, `props.theme.radius.*` 참조
- **provider.tsx**: 이미 존재하면 덮어쓰지 말고 내용을 비교 후 필요한 부분만 업데이트
- **styled.d.ts**: 이미 존재하면 import 경로 확인 후 필요 시 업데이트
- **빌드 성공**: 반드시 `npm run build` 실행 후 에러 없음 확인
