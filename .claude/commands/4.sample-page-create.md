토스 디자인 시스템(TDS)을 준수하면서 포트폴리오 샘플페이지를 자동으로 생성합니다.

⚠️ **사전 조건**: `/toss-theme-on` → `/content-setup` 순서대로 먼저 실행되어야 합니다.

## 사전 체크

아래 항목들을 확인하세요:

1. **디자인 토큰 상태 확인**
   - `portfolio/src/app/globals.css` — CSS 변수 (토스 컬러, 타이포그래피)
   - `portfolio/tailwind.config.ts` — Tailwind 설정
   - `/toss-theme-on` 명령어가 이미 실행되었는지 확인

2. **컴포넌트 인벤토리 확인**
   - `portfolio/src/components/ui/` — shadcn/ui 컴포넌트들
   - `portfolio/src/components/sections/` — 기존 섹션 컴포넌트 스타일 (참고용)

3. **프로젝트 데이터 확인**
   - `portfolio/src/data/` 또는 유사 위치의 프로젝트 데이터 파일 존재 여부

## Steps

### 1. 페이지 컴포넌트 생성
```
portfolio/src/app/[page-name]/page.tsx 생성
```
- 토스 디자인 토큰을 활용한 레이아웃
- 섹션 구조: Hero → About/Info → Projects → Contact
- Responsive 디자인 적용 (mobile, tablet, desktop)

### 2. 섹션별 컴포넌트 작성

#### 2-1. Hero Section
- 배경: `bg-white` 또는 `bg-gradient-to-br from-blue-50 to-white`
- 타이틀: `text-4xl md:text-5xl font-bold tracking-tight`
- 서브텍스트: `text-lg text-muted-foreground`
- CTA 버튼: 토스 블루 (`#3182F6`) primary 버튼

#### 2-2. About Section
- 배경: 교대 배경 `bg-muted` (극도로 옅은 회색 #F9FAFB)
- 카드 요소: `shadow-sm rounded-xl` 적용
- 콘텐츠: 프로필, 역량, 주요 경험 3~5개 항목

#### 2-3. Projects Section (선택)
- 프로젝트 카드 그리드: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`
- 각 카드: `shadow-sm hover:shadow-md transition-shadow rounded-xl p-6`
- 메타정보: 기술 스택은 badge로 표현

#### 2-4. Contact Section
- 배경: `bg-white`
- 간단한 문의 폼 또는 링크 (이메일, GitHub, LinkedIn 등)
- 버튼: Primary 액션 강조

### 3. 공통 스타일 가이드라인

**타이포그래피**
```
제목(h1): font-bold text-4xl tracking-tight
서브제목(h2): font-bold text-2xl tracking-tight
본문(p): font-normal text-base leading-relaxed
보조(small): font-normal text-sm text-muted-foreground
```

**스페이싱**
```
섹션 상하: py-12 md:py-20 lg:py-28
컨테이너 좌우: px-4 md:px-8 lg:px-12
요소 간격: gap-6 또는 gap-8
```

**컬러 팔레트**
```
Primary: #3182F6 (Toss Blue)
Background: #FFFFFF
Secondary Background: #F9FAFB
Text: #1B1C1F
Text Secondary: #4E5968 or #8B95A1
Border: #E5E8EB
```

**쉐도우**
```
Shadow-sm: 0 1px 4px rgba(0,0,0,0.06)
Shadow-md: 0 4px 16px rgba(0,0,0,0.08)
Shadow-lg: 0 8px 32px rgba(0,0,0,0.10)
```

### 4. 코드 구조 예시

```typescript
// portfolio/src/app/sample/page.tsx
import { HeroSection } from '@/components/sections/hero';
import { AboutSection } from '@/components/sections/about';
import { ProjectsSection } from '@/components/sections/projects';
import { ContactSection } from '@/components/sections/contact';

export default function SamplePage() {
  return (
    <main className="w-full">
      <HeroSection />
      <AboutSection />
      <ProjectsSection />
      <ContactSection />
    </main>
  );
}
```

### 5. 섹션 컴포넌트 생성 위치
- `portfolio/src/components/sections/hero.tsx`
- `portfolio/src/components/sections/about.tsx`
- `portfolio/src/components/sections/projects.tsx` (필요시)
- `portfolio/src/components/sections/contact.tsx`

### 6. 빌드 및 검증
```bash
cd portfolio && npm run build
npm run dev  # 로컬 확인
```

페이지가 정상 렌더링되고, 토스 디자인이 일관되게 적용되었는지 확인

### 7. 완료 보고
- 생성된 파일 목록
- 적용된 섹션
- 빌드 성공 여부

## Rules

- **디자인 일관성**: 토스 디자인 가이드(`toss-theme-on.md`)에 정의된 토큰만 사용
- **콘텐츠 보존**: 기존 프로젝트 데이터는 수정하지 않음 (읽기만 수행)
- **반응형**: 모든 섹션은 mobile-first 방식으로 개발
- **성능**: 이미지는 `next/image`로 최적화
- **접근성**: Semantic HTML 사용, 적절한 heading 계층 유지
- **빌드**: 생성 후 반드시 `npm run build` 실행하여 에러 확인
