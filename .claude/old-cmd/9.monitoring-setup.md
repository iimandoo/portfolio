#### 2-2. GA 컴포넌트 생성

**`src/components/google-analytics.tsx`** 생성:

```typescript
'use client';

import Script from 'next/script';

export function GoogleAnalytics({ measurementId }: { measurementId: string }) {
  return (
    <>
      <Script
        strategy="afterInteractive"
        src={`https://www.googletagmanager.com/gtag/js?id=${measurementId}`}
      />
      <Script
        id="google-analytics"
        strategy="afterInteractive"
        dangerouslySetInnerHTML={{
          __html: `
            window.dataLayer = window.dataLayer || [];
            function gtag(){dataLayer.push(arguments);}
            gtag('js', new Date());
            gtag('config', '${measurementId}', {
              page_path: window.location.pathname,
            });
          `,
        }}
      />
    </>
  );
}
```

### 4. Web Vitals 모니터링

**`src/lib/web-vitals.ts`** 생성:

```typescript
import { getCLS, getFID, getFCP, getLCP, getTTFB } from 'web-vitals';

export function reportWebVitals() {
  if (typeof window === 'undefined') return;

  // Core Web Vitals
  getLCP((metric) => {
    console.log('LCP:', metric.value, 'ms');
    sendMetric('web_vitals', {
      name: 'LCP',
      value: metric.value,
      rating: metric.rating,
    });
  });

  getCLS((metric) => {
    console.log('CLS:', metric.value);
    sendMetric('web_vitals', {
      name: 'CLS',
      value: metric.value,
      rating: metric.rating,
    });
  });

  getTTFB((metric) => {
    console.log('TTFB:', metric.value, 'ms');
    sendMetric('web_vitals', {
      name: 'TTFB',
      value: metric.value,
      rating: metric.rating,
    });
  });

  // FCP (First Contentful Paint)
  getFCP((metric) => {
    console.log('FCP:', metric.value, 'ms');
    sendMetric('web_vitals', {
      name: 'FCP',
      value: metric.value,
      rating: metric.rating,
    });
  });

  // FID (First Input Delay) - 웹 바이탈스 지표
  getFID((metric) => {
    console.log('FID:', metric.value, 'ms');
    sendMetric('web_vitals', {
      name: 'FID',
      value: metric.value,
      rating: metric.rating,
    });
  });
}

function sendMetric(name: string, data: any) {
  // Google Analytics로 전송
  if (typeof window !== 'undefined' && (window as any).gtag) {
    (window as any).gtag('event', name, data);
  }

  // API로 전송 (선택)
  // fetch('/api/metrics', { method: 'POST', body: JSON.stringify({ name, data }) });
}
```

**`src/app/layout.tsx`** 업데이트:

```typescript

export default function RootLayout({ children }: { children: React.ReactNode }) {
  useEffect(() => {
    reportWebVitals();
  }, []);

  return (
    // ...
  );
}
```

### 5. 이벤트 추적 설정 (선택)

**`src/lib/events.ts`** 생성:

```typescript
export function trackEvent(eventName: string, eventData?: Record<string, any>) {
  if (typeof window !== 'undefined' && (window as any).gtag) {
    (window as any).gtag('event', eventName, eventData);
  }
}

// 예시: 프로젝트 클릭 추적
export function trackProjectClick(projectId: string) {
  trackEvent('project_click', {
    project_id: projectId,
    timestamp: new Date().toISOString(),
  });
}

// 예시: 외부 링크 클릭
export function trackExternalLink(url: string) {
  trackEvent('external_link_click', {
    url,
    timestamp: new Date().toISOString(),
  });
}

// 예시: 페이지 스크롤 깊이
export function trackScrollDepth(depth: number) {
  trackEvent('scroll_depth', {
    depth_percent: Math.round(depth),
  });
}
```

### 6. 에러 로깅 설정 (선택)

#### 6-1. Sentry 통합 (권장)

```bash
npm install @sentry/nextjs
```

**`sentry.client.config.ts`** 생성:

```typescript
import * as Sentry from '@sentry/nextjs';

Sentry.init({
  dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
  environment: process.env.NODE_ENV,
  integrations: [
    Sentry.replayIntegration({
      maskAllText: true,
      blockAllMedia: true,
    }),
  ],
  tracesSampleRate: 1.0,
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,
});
```

**`sentry.server.config.ts`** 생성:

```typescript
import * as Sentry from '@sentry/nextjs';

Sentry.init({
  dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
  environment: process.env.NODE_ENV,
  tracesSampleRate: 1.0,
});
```

**`.env.local`** 업데이트:

```env
NEXT_PUBLIC_SENTRY_DSN=https://your-sentry-dsn@sentry.io/project-id
```

### 7. 성능 모니터링 대시보드

Vercel Dashboard에서 모니터링:

1. [Vercel Dashboard](https://vercel.com/dashboard)
2. 프로젝트 선택
3. "Analytics" 탭 클릭
4. 다음 지표 모니터링:
   - **Core Web Vitals**:
     - LCP (Largest Contentful Paint): < 2.5초
     - FID (First Input Delay): < 100ms
     - CLS (Cumulative Layout Shift): < 0.1
   - **Performance**:
     - Response Time
     - Edge Request Latency
     - Bandwidth

### 8. Google Analytics 대시보드 구성

#### 8-1. 핵심 리포트

1. Google Analytics 계정 접속
2. "리포트" → "생명 주기"
3. 다음 리포트 활용:
   - **획득**: 트래픽 출처 분석
   - **참여**: 사용자 행동 분석
   - **변환**: 목표 달성 추적

#### 8-2. 커스텀 리포트 생성

1. "탐색" → "자유형식" 클릭
2. 다음 Dimension & Metrics 추가:
   - **Dimensions**: 페이지 경로, 사용자 국가, 기기
   - **Metrics**: 사용자, 세션, 평균 참여 시간

## Rules

- **데이터 수집**: 개인정보 정책에 Google Analytics 명시 필수

- **리소스**: Vercel Analytics는 자동으로 Core Web Vitals 수집 (권장)
- **정기 검토**: 월 1회 이상 분석 데이터 검토
